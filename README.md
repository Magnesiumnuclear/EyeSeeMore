# 🖼️ EyeSeeMore (Alpha)

**EyeSeeMore** 是一個小巧的實驗性本地圖片搜尋工具。

當圖片收藏日益增加，而檔名往往是無意義的亂碼時，尋找特定圖片就成了一大難題。本專案的初衷是透過視覺模型讓電腦試著「理解」圖片內容，幫助使用者找回那些遺忘在角落的回憶。

> [!WARNING]
> 目前專案仍處於早期 **Alpha** 階段，許多功能尚在磨合與實驗中。

---

## ✨ 核心特色

* **不依賴檔名：** 嘗試透過視覺模型理解圖片內容，即便檔名完全沒有意義，也能透過描述文字進行檢索。
* **尊重隱私：** 所有的運算、分析與儲存都在本地端完成，圖片絕對不會被上傳到任何雲端伺服器。
* **開箱即用：** 極力簡化安裝流程，透過 **DirectML** 技術，讓各類顯卡（AMD、Intel、NVIDIA）都能輕鬆執行 AI 搜尋，無需手動配置複雜的 CUDA 環境。

## 💡 使用情境 (Use Cases)

* **瞬時社群分享：** 與朋友聊天時想發梗圖卻找不到？EyeSeeMore 旨在減少滑動尋找的流程，實現瞬間分享。
* **從截圖中尋找資訊：** 支援搜尋「錯誤代碼」或「會議記錄」，系統會透過 **OCR** 掃描圖片文字內容，精準定位重要截圖。
* **視覺筆記整理：** 直接搜尋書本頁面或手寫筆記照片中的關鍵詞，快速找回所需資料。
* **無懼亂碼檔名：** 通訊軟體下載的隨機數字檔名不再是障礙，無需手動重命名也能輕鬆檢索。

## ⌨️ 快捷鍵與操控

預覽模式提供類似遊戲的流暢巡覽體驗：

| 按鍵 | 功能說明 |
| :--- | :--- |
| `Space` | 快速開啟 / 關閉圖片預覽 |
| `W` / `A` / `S` / `D` | 預覽模式下快速切換與瀏覽圖片 |
| `Shift` | 即時切換 OCR 紅框標記，查看文字分佈 |

## 📝 如何啟用 OCR 文字辨識

EyeSeeMore 為了節省運算資源，預設不會對所有資料夾自動進行文字掃描。若您需要搜尋圖片內的文字內容，請務必執行以下步驟：
點擊介面左下角的 ⚙️ 設定 按鈕。
切換至 📁 資料夾管理 分頁。
在目標資料夾項目上點擊 右鍵。
選擇 「添加 XX OCR 標記」（例如：添加中文 OCR 標記）。
系統將會開始背景掃描，並在進度條顯示預估剩餘時間，補全該資料夾圖片的文字資訊。

## 🛠️ 技術架構

本專案採用混合架構，兼顧執行效率與開發彈性：

* **後端邏輯：** 基於 **Python** 開發。
* **推理引擎：** 採用 **ONNX Runtime**，透過 **DirectML** 實現硬體加速 (DirectX 12)。
* **GUI & 啟動器：** 透過 **C++** 撰寫小巧的啟動器與安裝程式，確保環境與路徑依賴的完整性。
* **離線運行：** 內建本地分詞資源 (**Tokenizer**)，確保在無網路環境下依然能正常啟動與搜尋。

## 📂 模型目錄結構 (Model Directory Structure)

為了確保程式正常啟動與運行，請確保您的 `models/` 資料夾遵循以下路徑格式。若為綠色安裝包，通常已內建此結構：

```text
models/
├── onnx_clip/               # CLIP 語意搜尋模型
│   ├── ViT-B-32_image.onnx
│   ├── ViT-B-32_text.onnx
│   ├── ViT-H-14_image.onnx
│   └── ... (其他支援的模型)
├── tokenizers/              # 離線分詞資源
│   ├── openai-clip/         # 標準模式用
│   └── xlm-roberta/         # 多語系模式用
└── ocr/                     # OCR 文字辨識模型包
    ├── common/              # 通用偵測模型
    ├── ch/                  # 中文語言包 (rec.onnx, dict.txt)
    ├── jp/                  # 日文語言包
    └── ... (其他語言)
