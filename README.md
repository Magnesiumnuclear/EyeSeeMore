# EyeSeeMore (Alpha)

EyeSeeMore 是一個小巧的實驗性本地圖片搜尋工具。

當圖片收藏日益增加，而檔名往往是無意義的亂碼時，尋找特定圖片就成了一大難題。EyeSeeMore 的初衷正是為了解決這個痛點：透過視覺模型讓電腦試著「理解」圖片內容，幫助使用者找回那些遺忘在角落的回憶。

> **注意：** 目前專案仍處於早期 Alpha 階段，許多功能尚在磨合與實驗中。

## 核心特色

* **不依賴檔名：** 嘗試透過視覺模型理解圖片內容，即便檔名完全沒有意義，也能透過描述文字進行檢索。
* **尊重隱私：** 所有的運算、分析與儲存都在本地端完成，圖片絕對不會被上傳到任何雲端伺服器。
* **開箱即用：** 極力簡化安裝流程，透過 DirectML 技術，讓各類顯卡（AMD、Intel、NVIDIA）都能輕鬆執行 AI 搜尋，無需手動配置複雜的 CUDA 環境。

## 使用情境 (Use Cases)

* **瞬時社群分享：** 與朋友聊天時，腦中浮現一張完美對應的梗圖，卻淹沒在數千張圖片中。EyeSeeMore 旨在減少滑動尋找的冗長流程，實現瞬間分享。
* **從截圖中尋找關鍵資訊：** 支援搜尋「錯誤代碼」或「會議記錄」，系統會嘗試透過 OCR 掃描圖片內的文字內容，精準定位重要的截圖。
* **視覺筆記整理：** 直接搜尋書本頁面或手寫筆記照片中的關鍵詞，快速找回所需資料。
* **無懼亂碼檔名：** 通訊軟體下載的圖片通常是一串隨機數字，現在無需手動重新命名，也能輕鬆檢索。

## 快捷鍵與操控

目前已實作的基礎操控功能，提供類似遊戲的流暢預覽體驗：

* `Space`：快速開啟 / 關閉圖片預覽。
* `W` / `A` / `S` / `D`：在預覽模式下快速切換與瀏覽圖片。
* `Shift`：即時切換 OCR 紅框標記，查看圖片中的文字分佈。

## 技術架構

* **硬體加速：** 採用 DirectML 技術，只要支援 DirectX 12 即可進行硬體加速，大幅降低使用門檻。
* **離線運行：** 內建本地分詞資源（Tokenizer），確保在無網路環境下依然能正常啟動與搜尋。
* **穩定部署：** 透過 C++ 撰寫的小巧啟動器與安裝程式，確保環境與模型的完整性，解決繁瑣的路徑依賴問題。

## 未來願景與 Roadmap

我們認為搜尋體驗不應侷限於電腦端，行動端的搜尋與分享同樣重要。

### 近期目標
- [ ] 支援更強大的 CLIP 模型，實現「文字 + 圖片」的複合式搜尋。
- [ ] 導入更強大的 OCR 模型，提供更細部的文字搜尋與編輯功能。
- [ ] 實作軟體內的虛擬資料夾功能。
- [ ] 開發虛擬資料夾的自動分類系統。

### 終極目標：自訂 Android 手機鍵盤 (Custom Keyboard)
這是我們對於行動端 UX 的最終極願景，旨在徹底消除分享時的摩擦感：
- [ ] 實作行動端 Web API 以支撐手機端的搜尋需求。
- [ ] 探索並開發 Android 自訂鍵盤擴充功能。
  * *目標：讓使用者在任何聊天軟體（如 LINE、Discord 等）中，無需切換 App，直接在輸入法介面打字搜尋，點擊結果即可貼上發送，讓 EyeSeeMore 成為手機輸入法的一部分。*
